** Basic TODO Functionality
Any headline becomes a TODO item when it starts with the word ‘TODO’.改变TODO的状态的时候也会改变tag的状态.查看变量 *org-todo-state-tags-triggers* .
1. C-c C-t (org-todo): 循环改变TODO的状态.如果TODO items可以使用快捷访问,那么会让你在minibuffer中快捷访问,如果 *org-use-fast-todo-selection* 不是nil的时候,这是默认的方式.当有多个组时,只在当前组上做状态改变.
2. S-RIGHT/LEFT: 下/上一个状态.如果有多个组,则会在所有状态上做循环.
3. C-c / t(org-show-todo-tree): 生成TODO items视图.把整个buffer折叠起来,只显示处理未完成状态.
4. C-c / T: 同第3点,只不过你可以使用关键字来搜索你想要查的TODO items
5. C-u C-u C-c C-t: 移动到下一个组.比如可以从TODO或者DONE状态移动到REPORT.也可以从REPORT/BUG/KNOWNCAUSE/FIXED等移动到CANCELED.
** Extended Use of TODO Keywords
*** org-todo-keywords as sequences
如下所示.其中'|'用来区分未完成状态和完成状态.如果没有'|',那么最后一个则被当作完成状态.
#+BEGIN_SRC emacs-lisp
  (setq org-todo-keywords
        '((sequence "TODO" "FEEDBACK" "VERIFY" "|" "DONE" "DELEGATED")))
#+END_SRC

#+RESULTS:

While using a list with face properties as shown for ‘CANCELED’ should work, this does not always seem to be the case. If necessary, defne a special face and use that. *** TODO-item's dependencies
TODO的依赖有两种情况.
1. 所有子item没有全完成done的时候,父item不能变成DONE状态. 设置 *org-enforce-todo-dependencies* 为t,开启第1类依赖.
2. 所有的兄item没有完成的时候,item本身不能标记为完成.  使用ORDERED property开启第2类依赖
3. 可以设置某些子item没有依赖.使用NOBLOCKING property来设置.
4. C-c C-x o(org-toggle-ordered-property): 开关ORDERED property. However, if you would like to track the value of this property with a tag for better visibility, customize the variable *org-track-ordered-property-with-tag*.
5. C-u C-u C-u C-c C-t: 忽略依赖,强制更新item的状态. If you set the variable org-agenda-dim-blocked-tasks, TODO entries that cannot be marked as done because of unmarked children are shown in a dimmed font or even made invisible in agenda views
6. You can also block changes of TODO states by using checkboxes. If you set the variable org-enforce-todo-checkbox-dependencies, an entry that has unchecked checkboxes is blocked from switching to DONE.

** Checkboxes
Every item in a plain list can be made into a checkbox by starting it with the string ‘[ ]’
Checkboxes work hierarchically, so if a checkbox item has children that are checkboxes, toggling one of the children checkboxes makes the parent checkbox reﬂect if none, some, or all of the children are checked.
If the current outline node has an ‘ORDERED’ property, checkboxes must be checked off in sequence, and an error is thrown if you try to check oﬀ a box while there are unchecked boxes above it.
1. C-c C-c (org-toggle-checkbox): 修改checkboxr状态
2. C-c C-x C-b (org-toggle-checkbox):
3. (org-insert-todo-heading): 插入一个checkbox.如果在一个headline行上,插入一个TODO headline.如果在一个plain list,就插入一个checkbox.
4. C-c #(org-update-statistics-cookies): 更新完成百分比.增加C-u则更新整个buffer. If you delete boxes/entries or add/change them by hand, use this command to get things back into sync.

** Property Inheritance
If the parent in a tree has a certain property, the children can inherit this property. Org mode does not turn this on by default.
However, if you fnd inheritance useful, you can turn it on by setting the variable org-use-property-inheritance. It may be set to t to make all properties inherited from the parent, to a list of properties that should be inherited, or to a regular expression that matches inherited properties.
If a property has the value nil, this is interpreted as an explicit un-defne of the property, so that inheritance search stops at this value and returns nil.
